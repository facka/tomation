Key Features

Where Matchers

Declarative Element Targeting

Define UI elements with readable selectors such as is.BUTTON.where(idIs('login-btn')).as('Submit') to keep tests explicit and maintainable.

const submitButton = is.BUTTON
  .where(idIs('login-btn'))
  .as('Submit')
Save To Context

Capture Dynamic Values for Later Steps

Store runtime values from the DOM and reuse them in later steps to keep multi-step flows robust and data-aware.

SaveText(orderIdLabel).as('orderId')
Type('{{ctx.orderId}}').in(searchInput)
